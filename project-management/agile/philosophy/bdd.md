# behavior driven development

- describes the behavior of the system from the *outside in*

- used for integration testing

- uses a single syntax to describe user stories and features scenarios that both
  stakeholders and developers can understand


### bdd feature scenarios

- As a [role]
- I need [some functionality]
- So that [some business value]

- scenario:
  - *given* a set of preconditions
  - *when* an event occurs
  - *then* some outcome is observed
