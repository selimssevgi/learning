# scrum vs kanban

- cadence
- release methodology
- key metrics
- change philoshopy

## scrum

- fixed length sprint
- end of each sprint
- product owner, scrum master, development team
- velocity
- teams should strive not to make change to the sprint forecast during sprint

## kanban

- continuous flow
- continuous delivery
- no existing roles (optional agile coach)
- cycle time
- change can happen at any time
